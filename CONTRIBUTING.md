# How to Contribute

1. Consider grouping articles into folders.
2. The top-level directory should include only high-level concepts.
3. Use `-` when naming files and folders with spaces.
4. Use a `.order` file to specify the order of articles in a given folder.
5. Link to readme files in other repositories that might contain more detailed information.
6. Use a table of contents which will use Markdown headings on each article.
7. Use Markdown headings such as `##` to break out sections in the article.
8. Each article should include an `## Overview` section.
9. Each article should include a Table of Content at the top. Use the syntax `[[_TOC_]]`.
10. All images will be stored under the `\.attachments` directory. This is only visible from the `git` repository view and can be managed using `Visual Studio Code`.
